Mapping rovio's position in a room

This part of my project is a pretty interesting one. I wanted to track the movement of the rovio in a room using the true track system. Rovio knows its own position in the room but we don't have an idea of the presence of rovio in the room unless we see it directly by ourselves. My idea was to create a map of rovio's movement with the help of the coordinates of the true track system. The API is good enough to give you the coordinates by means of the status report cgi script of rovio.

I call this cgi script from MATLAB to get the overall status of rovio. Then I filter out all the unwanted components as i just need the x and y coordinates of Rovio. Before doing this we have to calibrate the value returned by the north star sensor since its a dimensionless quantity. For this I made rovio to wander around and got the x,y values and measured the actual distance that rovio covered using measurement tape. Finally I found that 55 units of the north star sensor value is equivalent to 1 centimeter. Another pre-requesite is we need to align the two stars in parallel to the detector in rovio for easy calibration

Once we have calibrated the system, we need to start recording these values. I get the x and y values in matlab and store it in a vector. Then I write this vector to an external file (excel or txt) as two columns. I do this for a particular amount of time where the rovio keeps on moving. I write this value every 1 second. I have function called rovio_home in matlab which will calculate the home position of rovio by averaging 10 values of x and y when rovio is static. I do a smoothing since these sensor values are prone to change even if rovio stays at a particular location. From the data file I use a real time plotting software to pick up these values and plot x against y which gives you the map. The data plotting software again picks up the value from data file every second. Thus the system remains synchronized. This can be used for various purposes such as to calculate the dimensions of room by allowing rovio to move around the corners of the room. We can also measure the dimensions of an obstacle. This can be useful in calculating the shortest path, the application goes on... This is the video of the real time plotting of rovio's movement. Now having a look at this you know how rovio is moving without actually seeing it
